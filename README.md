# TikTok Publisher

Веб-панель для публикации собственных коротких видео (со своим вотермарком) в несколько TikTok-аккаунтов через открытую библиотеку [tiktok-uploader](https://github.com/wkaisertexas/tiktok-uploader) (браузерная автоматизация на Playwright, аутентификация по cookies).

## Запуск

1. Скопируйте `.env.example` в `.env` и заполните:
   - `SECRET_KEY` — случайная строка для шифрования cookies аккаунтов в БД.
   - `PANEL_USERNAME` / `PANEL_PASSWORD` — доступ к самой панели.
2. `docker-compose up --build` (образ ставит Playwright + Chromium).
3. Открыть `http://localhost:8000` (Basic Auth — логин/пароль из `.env`).
4. Для каждого TikTok-аккаунта: войти в него в своём браузере, экспортировать `cookies.txt` (например, расширением "Get cookies.txt"), загрузить файл в панели на странице аккаунта.

## Важно

- Публикация идёт через имитацию залогиненной browser-сессии, а не через официальный API — TikTok может детектировать автоматизацию (капча, временная блокировка). Не публикуйте слишком часто/одновременно во все аккаунты, используйте отдельные cookies на разные сессии.
- Никакого автоматического логина по паролю нет — только cookies от уже выполненного пользователем входа.
- Загружайте только видео, на которые у вас есть права.

## Структура

См. `backend/app/` — `api/` (роуты), `auth/tiktok_oauth.py` (шифрованное хранение cookies аккаунтов), `video/` (загрузка + ffmpeg-вотермарк), `publisher/tiktok_client.py` (обёртка над tiktok-uploader), `scheduler.py` (очередь публикаций на APScheduler).
